The Internet and the Web

Origins of the Internet

  • ARPANET introduced by DARPA in 1960s [pdf]
  • A national network for crisis communication
  • Shared mainframe resources
  • Development of communication protocols (TCP/IP)
  • Gave way to NSFNet in 1980s

Origins of the Web

  • Proposed by (Sir) Tim-Berners Lee, a computer scientist at CERN, in 1989
  • Based on the concept of hypertext — the ability to access multiple levels of information through links
  • Unveiled in 1991 on USENET
  • Utilizes the connections provided by the Internet

World Wide Web

URL
uniform resource locator
URI
uniform resource identifier
HTML
hypertext markup language; structures content for the Web
HTTP
hypertext transfer protocol; how clients, servers exchange information

The first Web site

http://info.cern.ch

The Web grows up

  • Mosaic browser introduced by NCSA's Marc Andreessen in January 1993
    • First browser to support inline images
  • Software developed for Windows, Mac
  • Web adoption grows
    • November 1992: 26 web sites
    • June 1993: 130 sites
    • December 1993: 632 sites

The Browser Wars

  • 1994: Netscape introduced
  • 1995: Internet Explorer introduced
  • Both included proprietary tags, scripting languages for coding web sites
  • Resulted in campaigns to standardize the web

Web Standards

  • Developed by the W3C to ensure compatibility across browsers, devices
  • Assures the web is backward and foward compatible
  • Current standards: HTML5, CSS3, ECMAScript 5 (a.k.a Javascript)
  • W3C Validator can help root out non-standard code

Understanding browsers

  • Each browser has a different 'rendering' engine
  • Most modern browsers follow standards
  • Standards are usually implemented as they are approved by the W3C
    • But ... which feature and how quickly it gets added to a browser depends on its creator

Web Basics

Tools of the trade

  • A Web browser
  • A text editor
    • Atom
    • Brackets
    • Sublime Text
  • Multimedia suite (i.e. Adobe Creative Suite)
  • A server for hosting

Text editors vs. word processors

  • Text editors work with plain text
    • ASCII Standard
    • Binary glyphs (letters, numbers, punctuation)
    • 1110111 1100101 1100010 = web
  • Word processors store plain text, styling, layout in a binary package which is not readable by Web browsers
  • DO NOT use a word processor for coding

How the web works

  • Each 'node' has an IP (Internet protocol) address
    • a node can be a computer, server, phone, etc.
  • 232 (IPv4) or 2128 (IPv6) possible combinations
  • What is your IP address?
    • your location as it relates to the Internet and geographically

Domain Name System (DNS)

  • The 'phonebook' of the Internet
  • Each domain is mapped to an IP address
    • google.com = 173.194.37.102
    • umiami.edu = 129.171.32.100
  • A DNS server routes you to the correct IP address
    • includes web, mail, secure connections, etc.

Anatomy of a domain name

http://www.umiami.edu

  • edu = top-level domain
  • umiami = second-level or ... name
    • we can 'reserve' these for our own use
  • www = subdomain
    • could point to a different part, directory of a server

Top-level domains

  • generic
    • .com: commercial
    • .net: network
    • .org: organization
    • .edu: educational
    • .gov: government
    • .mil: military
  • country
    • .us: United States
    • .ca: Canada
    • .de: Germany
    • some countries limit access to citizens

Finding a domain

  • WHOIS
    • Every domain registration collects contact information
    • Personal info can be protected (usually for a fee)
    • To see who a domain belongs to, type: whois <domain> at the command line
  • http://domai.nr helps you find unique combinations

Domain name tips

  • Make it easy to remember
  • Keep it short (8 characters is great!)
  • Try for a .com, but other top-level domains are okay
  • Avoid hyphens, numbers if possible
  • Avoid copyright, trademark infringement
